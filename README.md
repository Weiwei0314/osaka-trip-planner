# osaka-trip-planner（旅行行程表）

離線可用的旅行行程規劃 PWA（Progressive Web App）。沒有後端、沒有建置流程，資料全部存在瀏覽器本機（`localStorage`），可以安裝到手機主畫面當作一般 App 使用。

## 功能

- **多行程管理**：可同時建立多個行程，列表會顯示狀態（即將出發／旅行中／已結束）、倒數天數、行程項目數與未完成代辦數。
- **總覽**：進入行程後預設顯示的分頁。顯示出發倒數／旅行中狀態、行程項目數、代辦完成度、購物預估金額，若今天正好在旅行日期範圍內會列出今日行程，並列出未完成的代辦事項。
- **時間軸**：依行程天數分組顯示行程項目，可設定分類（景點／餐廳／交通／住宿／購物／其他）、時間（輪盤選擇器）、地點、備註、最多 20 張照片；支援拖曳調整順序／換天／與其他項目互換時段；可在每個行程項目下掛購物、代辦子項目。
- **購物清單**：品項、數量、金額、分類、所屬天數、備註、照片，自動加總已購買數量與預估金額。
- **代辦事項**：事項、截止日期、優先度（高／中／低）、所屬天數、備註、照片，依優先度排序。
- **複製到其他行程**：行程項目、購物、代辦事項都可以複製一份到另一個行程，複製後兩邊資料各自獨立、互不影響。
- **CSV 匯入**：進入行程詳細頁後，右上角的文件圖示可匯入 CSV 檔案，一次匯入多筆行程項目／購物／代辦／地點收藏（格式見下方）。
- **JSON 匯出／匯入**：在行程列表可以把單一行程（含所有子項目）匯出成 JSON 檔分享，或用左上角「匯入」按鈕匯入別人分享的行程檔案（會建立成一個全新行程）。
- **離線支援**：透過 Service Worker（`sw.js`）快取靜態資源，離線也能開啟使用。

## 如何執行

純靜態網頁，沒有 npm 套件、沒有建置步驟：

- 直接用瀏覽器開啟 `home.html`，或
- 用任意靜態伺服器在本機啟動（例如 `npx serve .`），透過 `http://localhost` 開啟——若要測試「安裝到主畫面」或離線快取（Service Worker），需要走 `http://` 或 `https://`，不能是 `file://`。

## 檔案結構

| 檔案 | 說明 |
|---|---|
| `home.html` | 唯一的頁面，所有畫面（行程列表／時間軸／購物／代辦／總覽）都是同一個頁面裡切換 |
| `app.js` | 全部邏輯：資料層（`DB`）、渲染、CSV/JSON 匯入匯出、時間輪盤選擇器、拖曳排序 |
| `style.css` | 樣式 |
| `sw.js` | Service Worker，離線快取用；改動 `home.html`／`app.js`／`style.css` 後記得同步調高 `CACHE` 版本號，否則已安裝的使用者會讀到舊快取 |
| `manifest.json` | PWA 設定（圖示、名稱、standalone 模式） |
| `icon.svg` | App 圖示 |

## 資料儲存與限制

行程／事件／購物／代辦這些文字資料存在瀏覽器的 `localStorage`（每個網站通常只有 5–10MB 硬性配額）。**照片則存在 `IndexedDB`**（`trip-planner-photos` 資料庫），以原始二進位 Blob 儲存、不經過 Base64 膨脹，配額比 `localStorage` 大得多（通常是裝置可用空間的一個比例），兩者互相獨立，照片不會再擠壓到文字資料的配額。目前每個項目最多 20 張照片（`MAX_PHOTOS_PER_ITEM`），上傳時會自動壓縮到最長邊 1600px、JPEG 品質 0.8。

實作重點（如果要繼續修改這部分程式碼）：
- 每個項目的 `photos` 欄位只存照片在 IndexedDB 裡的 id，不是圖片本身；讀取時要透過 `getPhotoUrl(id)`（非同步，內部用 `URL.createObjectURL` 快取）換成可以放進 `<img src>` 的網址。
- 因此所有會顯示照片的渲染函式（`photoThumbsHtml`、`eventItem`、`shopItem`、`todoItem`、`renderTimeline`、`renderShopping`、`renderTodo`、`renderOverview`、`renderDetail`、`render`、`renderPhotoStrip`）都是 `async`。
- 刪除項目／行程、在編輯時移除照片、或另存副本時，都要記得同步清掉 IndexedDB 裡對應的 Blob（見 `deletePhotoBlobs`／`duplicatePhotoBlobs`），避免留下孤兒資料或兩份資料共用同一張照片。
- JSON 匯出／匯入會在檔案邊界把照片轉回／轉出 Base64（`inlinePhotosForExport`／`importPhotosFromDataUrls`），確保匯出的 JSON 檔仍然是自包含、可攜的單一檔案。
- 第一次載入時會自動把舊版本（照片直接存 Base64 在 localStorage）的資料搬進 IndexedDB（`migratePhotosToIndexedDB`），只會執行一次。

## CSV 匯入格式

標頭列（順序不限）：

```
類型,標題,日期,開始時間,結束時間,分類,地點,數量,金額,截止日期,優先度,備註
```

| 欄位 | 說明 |
|---|---|
| 類型（必填） | `行程`/`event`、`購物`/`shopping`、`代辦`/`todo`、`地點`/`location` |
| 標題（必填） | 項目名稱 |
| 日期 | `YYYY-MM-DD` 或 `YYYY/MM/DD`；也接受 Excel 日期序列數字。行程/購物用作所屬日期，代辦用作排入哪一天 |
| 開始時間／結束時間 | 只有「行程」項目會用，`HH:MM` 格式（也接受帶 AM/PM 的格式） |
| 分類 | 行程/地點用：景點、餐廳、交通、住宿、購物、其他；購物則是自由文字 |
| 優先度 | 只有「代辦」會用：高/中/低 |
| 備註 | 各類型通用 |

不需要的欄位留空即可。用 Excel 存檔時建議選「**CSV UTF-8（逗號分隔）**」而不是一般的「CSV（逗號分隔）」，否則中文字會因編碼不同變成亂碼。

## 更新紀錄

- 照片改存 `IndexedDB`（原本是 Base64 文字塞進 `localStorage`），不再共用文字資料的 5–10MB 配額
